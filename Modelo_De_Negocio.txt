MODELO DE NEGOCIO — Documentation Hub
======================================
Última actualización: 11/05/2026


────────────────────────────────────────────────────────────────
PROPUESTA DE VALOR
────────────────────────────────────────────────────────────────

Documentation Hub ayuda a empresas a mantener su documentación interna
coherente y consultable. El sistema:

1. Responde preguntas sobre la documentación en segundos (RAG)
2. Detecta contradicciones, duplicados e inconsistencias automáticamente
3. Propone mejoras concretas que el usuario puede aplicar en un clic
4. Muestra estadísticas de uso: qué se consulta, qué no, qué porcentaje
   del corpus está siendo realmente aprovechado

Público objetivo: equipos de conocimiento, áreas de calidad, RRHH, legal,
y cualquier equipo que gestione documentación normativa o de procesos.


────────────────────────────────────────────────────────────────
MODELO DE CRÉDITOS
────────────────────────────────────────────────────────────────

El sistema usa créditos prepago por operación. Cada organización tiene
un saldo de créditos que se consume con cada acción de IA.

Costes por operación
  Consulta de chat (/api/ask)              : 1 crédito
  Análisis de calidad rápido               : 5 créditos
  Análisis de calidad exhaustivo           : 20 créditos
  Análisis de estilo (/api/analyze-style)  : 2 créditos
  Mejora de documento (/api/improve)       : 1 crédito
  Subida de documento (indexación)         : GRATIS
  Resolución de duplicidades               : GRATIS (desde 11/05/2026,
                                             propuesta generada localmente)

Nota: el crédito se descuenta ANTES de ejecutar la operación para evitar
sobrepasar el límite con solicitudes paralelas (RPC atómica en Supabase).


────────────────────────────────────────────────────────────────
ESTRUCTURA DE PLANES (REFERENCIA)
────────────────────────────────────────────────────────────────

Los planes se gestionan vía Stripe. La estructura exacta de precios se
define en el dashboard de Stripe, no en el código.

Características comunes a todos los planes
  - Multi-tenant: varias organizaciones independientes
  - Subida de documentos sin límite de almacenamiento (limitado por Pinecone)
  - Chat RAG con historial de conversación
  - Panel de analítica para admins

Diferenciadores esperados por plan
  - Número de créditos mensuales incluidos
  - Número de miembros por organización
  - Acceso a análisis exhaustivo (alto coste en créditos)
  - Soporte prioritario

Créditos extra
  - Los usuarios pueden comprar créditos adicionales independientemente
    del plan vía /api/billing/buy-credits
  - Acumulables con los créditos del plan
  - No caducan (extra_credits separados de credits del plan)

Grace period
  - 90 días tras cancelación de suscripción antes de purgar datos
  - Tabla organizations: canceled_at + grace_period_ends_at
  - Durante grace period el acceso sigue activo (datos no eliminados)


────────────────────────────────────────────────────────────────
FLUJO DE FACTURACIÓN IMPLEMENTADO
────────────────────────────────────────────────────────────────

1. Usuario accede a /settings/billing
2. Elige plan → checkout Stripe → webhook actualiza organizations en Supabase
3. Webhook de Stripe gestiona:
   - checkout.session.completed → activar plan, asignar créditos
   - customer.subscription.updated → cambio de plan
   - customer.subscription.deleted → marcar canceled_at, iniciar grace period
   - invoice.payment_failed → notificar al admin

Compra de créditos extra
  - Pago único vía Stripe (no suscripción)
  - Webhook añade a organizations.extra_credits


────────────────────────────────────────────────────────────────
COSTES DE INFRAESTRUCTURA (ESTIMADOS)
────────────────────────────────────────────────────────────────

Pinecone serverless
  - Gratis: hasta 2GB storage, 100K vectores
  - Un documento de ~10 páginas genera ~10 vectores (1024 dims)
  - 100K vectores ≈ 10.000 documentos de tamaño medio

Supabase
  - Gratis: 500MB DB, 50K usuarios auth
  - Pro ($25/mes): 8GB DB, backups automáticos

Vercel
  - Gratis: 100GB bandwidth/mes, funciones serverless con timeout 30s
  - Pro ($20/mes): timeout hasta 300s (necesario para análisis largos),
    dominio personalizado

Anthropic (Claude)
  - Claude Haiku 4.5: ~$0.25 / M tokens input, ~$1.25 / M tokens output
  - Claude Sonnet 4.6: ~$3 / M tokens input, ~$15 / M tokens output
  - Un análisis rápido típico: ~15K tokens → ~$0.05 con Haiku, ~$0.10 con Sonnet
  - Un análisis exhaustivo: ~50K tokens → ~$0.15 Haiku + $0.50 Sonnet

Estimación para 100 usuarios activos / mes
  - 500 consultas de chat: ~$0.25 (Haiku)
  - 100 análisis rápidos: ~$10
  - 20 análisis exhaustivos: ~$14
  - Total API: ~$25/mes
  - Infraestructura (Vercel Pro + Supabase Pro): ~$45/mes
  - Total estimado: ~$70/mes para 100 usuarios activos
